# Data Model: Radio

## Сущность: Radio (компонент)

Представляет **один вариант** взаимоисключающего набора: визуально «круг + подпись», логически выбран или нет.

### Поля (логическая модель / пропсы)

| Поле | Описание | Правила |
|------|----------|---------|
| `checked` / `defaultChecked` | Выбран ли этот вариант | Controlled vs uncontrolled — как у нативного `input[type=radio]` в группе с одним `name` |
| `disabled` | Блокировка выбора | Не участвует в смене значения; **hover не меняет визуал** |
| `error` | Режим ошибки (invalid) | Визуал invalid + `aria-invalid` |
| `size` | Размер | `large` \| `medium` \| `small` |
| `name` | Имя группы | Одинаковое у всех радио одной группы (HTML) |
| `value` | Значение варианта | Строка (или как у нативного input) для идентификации при `onChange` |
| `id` | id инпута | Для связи с внешним label при композиции снаружи |
| `onChange` | Смена выбора | `ChangeEvent<HTMLInputElement>` |
| `label` / `children` | Текст справа от индикатора | Опционально; как у Checkbox для макета «индикатор + Text» |
| `className` | Доп. класс на корень | Не ломает модификаторы состояний |

### Проброс нативных атрибутов

- Разумный подмножество **`React.InputHTMLAttributes<HTMLInputElement>`** для `type="radio"`: `aria-*`, `data-*`, `required`, и т.д., с фиксированным **`type="radio"`**.

### Состояния отображения

| Состояние | Условие |
|-----------|---------|
| default | Не disabled, не error, без hover/focus |
| hover | Наведение на интерактивную зону, **не** disabled |
| focus | `:focus-visible` / кольцо по макету |
| disabled | `disabled === true` |
| invalid | `error === true` |

### Значения визуала (индикатор)

| Значение | Условие |
|----------|---------|
| unchecked | `checked !== true` (в controlled — по пропу) |
| checked | `checked === true` |

### Связи

- **Группа**: несколько `Radio` с общим `name`; ровно один с `checked` в controlled-режиме обеспечивает родитель.
- Текст ошибки формы — **вне** компонента (композиция).
- Противоречивое состояние «два checked с одним name» в controlled — ответственность потребителя; компонент отображает переданные пропы.
