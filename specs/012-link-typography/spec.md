# Feature Specification: Link (typography)

**Feature Branch**: `012-link-typography`
**Created**: 2026-05-11
**Status**: Draft
**Input**: User description: "Теперь давай создадим компонент Link по аналогии c https://tech.skbkontur.ru/kontur-ui/?path=/docs/react-ui_button-link--docs, но в парадигме нашей системы и ее принципов. Не делаем блоки, Доступности, Адаптивности. Используем исключительно стили проекта. Link — это часть typography, а не Button. В конце собери документацию по шаблону проекта."

## Описание

`Link` — текстовая ссылка как часть типографики проекта, не кнопка. Используется внутри абзацев, заголовков, ячеек таблиц, элементов списков и любого другого набора текста: ссылка наследует размер шрифта и межстрочный интервал у родителя и встаёт «в строку», не ломая ритм текста. Семантика и поведение — как у обычной ссылки/типографической кнопки: переход по адресу (`href`) или действие по клику (`onClick`), с состояниями `default`, `hover`, `active`, `focus-visible`, `disabled` и набором цветовых вариантов из палитры проекта.

`Link` MUST использовать только design tokens и шрифт `ONY One`. Никаких hardcoded цветов, размеров шрифта или произвольных стилей.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Ссылка внутри текста абзаца (Priority: P1)

Разработчик добавляет ссылку прямо в текст абзаца («Подробнее в [нашей документации].»). `Link` встаёт inline в строке, наследует размер шрифта и line-height окружающего текста, выглядит как часть этого текста и отличается только цветом и подчёркиванием.

**Why this priority**: основной сценарий ссылок в продуктовых интерфейсах — встраивание в наборный текст. Без корректного inline-поведения и наследования типографики компонент бесполезен в реальных макетах.

**Independent Test**: рендер `<p>Подробнее в <Link href="/docs">нашей документации</Link>.</p>` в Storybook/тесте. Размер шрифта, семейство и высота строки совпадают с родительским `p`; ссылка не ломает baseline; перенос длинного текста ссылки на следующую строку работает по словам.

**Acceptance Scenarios**:

1. **Given** `Link` находится внутри `<p>` с типографикой `Label/medium`, **When** компонент отрендерен, **Then** `font-family`, `font-size`, `line-height` ссылки совпадают со значениями родителя (наследование, а не явные значения).
2. **Given** длинный текст ссылки в узком контейнере, **When** контейнер сужается, **Then** текст ссылки переносится по словам, baseline не сбивается.
3. **Given** в одном абзаце соседствуют обычный текст и `Link`, **When** компонент отрендерен, **Then** между обычным текстом и текстом ссылки нет визуального «прыжка» по вертикали (одинаковая высота строки и baseline).
4. **Given** `Link` стоит внутри заголовка (`<h2>`), **When** компонент отрендерен, **Then** он наследует типографику заголовка (а не базовый текст), без явных переопределений размера.

### User Story 2 — Навигация по `href` и действие по `onClick` (Priority: P1)

Разработчик использует `Link` либо как ссылку с переходом (`href`), либо как inline-действие в тексте (`onClick` без `href`) — единый визуальный язык для «текстовых» действий и навигации.

**Why this priority**: без `href` и `onClick` компонент не может ни вести по адресу, ни запускать действие — то есть не выполняет свою главную функцию.

**Independent Test**: рендер `<Link href="https://example.com" target="_blank">Внешняя</Link>` и `<Link onClick={fn}>Действие</Link>`. По клику первая открывает адрес, вторая вызывает `fn`. С `disabled` — ни перехода, ни вызова.

**Acceptance Scenarios**:

1. **Given** `<Link href="/page">Перейти</Link>`, **When** пользователь кликает по ссылке, **Then** браузер переходит по адресу `/page`.
2. **Given** `<Link href="https://x.com" target="_blank">Внешняя</Link>`, **When** компонент отрендерен, **Then** к нативному элементу применяются атрибуты `target="_blank"` и `rel` с защитой от opener-эксплуатации (`noopener`).
3. **Given** `<Link onClick={fn}>Действие</Link>` без `href`, **When** пользователь кликает, **Then** вызывается `fn(event)` и не происходит навигации.
4. **Given** `<Link href="/x" onClick={fn}>Ссылка</Link>`, **When** пользователь кликает, **Then** сначала вызывается `fn`; если в `fn` вызван `event.preventDefault()`, навигация не происходит.
5. **Given** `<Link disabled href="/x" onClick={fn}>X</Link>`, **When** пользователь кликает, **Then** ни `fn`, ни навигация не выполняются; курсор — `not-allowed`.

### User Story 3 — Цветовые варианты и состояния (Priority: P1)

Дизайнер использует `Link` в нескольких ролях: основной акцент в тексте, второстепенный («чёрный» вариант для тех мест, где не нужен бренд-цвет), деструктивный (например, «Удалить» в строке таблицы). Состояния `hover`, `active`, `focus-visible`, `disabled` единообразны и берутся из токенов.

**Why this priority**: разные роли ссылки в одном интерфейсе нужны постоянно (бренд-CTA в тексте vs. «второстепенная» ссылка в табличке vs. «опасная» ссылка). Без вариантов компонент придётся форкать в каждом проекте.

**Independent Test**: рендер `<Link>` в трёх вариантах (`default`, `secondary`, `danger`) и в каждом состоянии (`default`, `hover`, `active`, `focus-visible`, `disabled`). Цвет и подчёркивание соответствуют токенам соответствующего варианта/состояния.

**Acceptance Scenarios**:

1. **Given** `<Link variant="default">…</Link>` в состоянии покоя, **When** компонент отрендерен, **Then** цвет текста — бренд-цвет из токенов (например, `--content-brand`), текст подчёркнут.
2. **Given** `<Link variant="secondary">…</Link>` в состоянии покоя, **When** компонент отрендерен, **Then** цвет текста — нейтральный из токенов (например, `--content-primary`), текст подчёркнут.
3. **Given** `<Link variant="danger">…</Link>` в состоянии покоя, **When** компонент отрендерен, **Then** цвет текста — деструктивный из токенов (например, `--content-error`), текст подчёркнут.
4. **Given** `<Link>` в любом варианте, **When** наведена мышь, **Then** цвет/подчёркивание меняются по правилам токенов hover-состояния (например, осветление/затемнение или смена `content-*-hover`).
5. **Given** `<Link>` в фокусе с клавиатуры, **When** установлен `:focus-visible`, **Then** появляется фокус-кольцо или контурный стиль, заданный токеном `--focus-ring`/аналогом проекта.
6. **Given** `<Link disabled>`, **When** компонент отрендерен, **Then** цвет — `--content-disabled` (или эквивалент), текст не подчёркнут (или подчёркивание приглушённое), курсор `not-allowed`, `hover`/`active` не меняют вид.

### User Story 4 — Иконки в ссылке (Priority: P2)

Разработчик добавляет к тексту ссылки иконку слева или справа (`startIcon`, `endIcon`) — например, стрелку «открыть в новой вкладке» или иконку документа. Иконка наследует цвет текста ссылки и масштабируется относительно текущего размера шрифта (`1em` или `currentColor`).

**Why this priority**: иконки рядом со ссылкой повышают читаемость и сигнализируют тип действия, но базовый компонент без иконок остаётся пригодным к использованию — это расширение, не блокатор MVP.

**Independent Test**: рендер `<Link startIcon={<Icon …/>}>Скачать</Link>` и `<Link endIcon={<Icon …/>}>Открыть</Link>`. Иконка вертикально центрируется по line-height текста, размер — относительно текста (`1em`), цвет — `currentColor`.

**Acceptance Scenarios**:

1. **Given** `<Link startIcon={<Icon name="download" />}>Скачать</Link>`, **When** компонент отрендерен, **Then** иконка стоит слева от текста, имеет квадратные пропорции, размер пропорционален текущему `font-size`.
2. **Given** `<Link endIcon={<Icon name="external" />}>Открыть</Link>`, **When** компонент отрендерен, **Then** иконка стоит справа от текста, цвет иконки совпадает с цветом текста (через `currentColor`).
3. **Given** `<Link variant="danger" startIcon={<Icon … />}>…</Link>`, **When** компонент отрендерен, **Then** иконка перекрашивается в тот же `--content-error`, что и текст.
4. **Given** `<Link disabled startIcon={<Icon … />}>…</Link>`, **When** компонент отрендерен, **Then** иконка перекрашивается в цвет disabled вместе с текстом.

### User Story 5 — Storybook и страница документации по шаблону проекта (Priority: P3)

Разработчик/дизайнер открывает Storybook → `Components → Link` и видит живые примеры всех вариантов и состояний, а также страницу документации, собранную по тому же шаблону, что и у `Input`, `Button`, `Checkbox` и других компонентов проекта.

**Why this priority**: документация и stories помогают распространению компонента, но API можно использовать и без них; не блокирует MVP.

**Independent Test**: в Storybook появилась страница `Components → Link`, на ней — список stories (default-вариант, secondary, danger, with-icons, disabled, в составе абзаца, в составе заголовка) и docs-страница, на которой нет блоков «Доступность» и «Адаптивность», есть «Подключение в проекте», «Пропсы», «Варианты», «Состояния», «Иконки», и базовое описание короткое (одна-две фразы — главная функция компонента).

**Acceptance Scenarios**:

1. **Given** запущен Storybook, **When** разработчик открывает `Components → Link`, **Then** видит stories для всех вариантов и состояний, а также docs-страницу.
2. **Given** docs-страница `Link`, **When** разработчик её просматривает, **Then** в ней есть кнопки `Figma` и `GitHub` сверху, краткое базовое описание (1–2 фразы), раздел `Подключение в проекте` с импортом, раздел `Пропсы`, раздел `Варианты` и раздел `Состояния` (+ опционально `Иконки`). Разделов «Доступность» и «Адаптивность» нет.
3. **Given** docs-страница `Link`, **When** разработчик ищет упоминания hardcoded цветов или собственного шрифта, **Then** ничего не находит: все примеры и preview работают через токены и `ONY One`.

### Edge Cases

- **Пустой `children`**: если у `Link` нет текста и нет иконок — компонент не рендерится (или рендерится как пустой span без побочных эффектов); проп `aria-label` НЕ предписывается (вне scope, доступность отдельным треком).
- **`disabled` + `href`**: атрибут `href` НЕ применяется к DOM-элементу или применяется без эффекта; клик не вызывает навигации; курсор `not-allowed`.
- **Длинная ссылка без пробелов** (например, URL): переносится по умолчанию по словам; для URL допустим `overflow-wrap: anywhere` — допущение, фиксируется в plan.
- **`Link` внутри `Button`**: запрещённое использование (вложенные интерактивные элементы); упоминается в документации, валидация в коде не требуется.
- **Несколько `Link` подряд в одной строке**: не вызывает «склеивания» (расстояние/обводка фокуса не наезжают друг на друга).
- **Глобальные стили `a { color: blue }` в окружении**: `Link` использует собственные токены и переопределяет наследованный `color` родительского `a`-стиля темы хоста (изоляция через `scopeClassName`/`turbo-ui-scope`, как у других компонентов).
- **Контекст с другим `font-family`** (нестандартный шрифт хоста): `Link` сохраняет `font-family: var(--family-brand, "ONY ONE", …)` или наследует — фиксируется в plan как осознанное решение (по умолчанию — наследование, чтобы Link был частью typography родителя; внутри `turbo-ui-scope` шрифт уже `ONY One`).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Библиотека MUST предоставлять компонент `Link` как часть typography (не Button). Компонент рендерится как inline-элемент и наследует `font-family`, `font-size`, `line-height`, `letter-spacing` родительского текста.
- **FR-002**: `Link` MUST поддерживать проп `href?: string`. При наличии `href` корневым DOM-элементом MUST быть `<a>`; без `href` — `<button type="button">` без рамки/фона (типографическая кнопка).
- **FR-003**: `Link` MUST поддерживать проп `onClick?: (event) => void`. Обработчик вызывается как для варианта с `href`, так и без; если в `onClick` вызван `event.preventDefault()`, навигация (для варианта с `href`) НЕ выполняется.
- **FR-004**: `Link` MUST поддерживать пропы `target` и `rel` для случаев с `href`. При `target="_blank"` без явного `rel` компонент автоматически добавляет `rel="noopener noreferrer"`.
- **FR-005**: `Link` MUST поддерживать проп `variant: "default" | "secondary" | "danger"` (имена ориентировочные, финальные — в plan). Цвет текста и подчёркивания каждого варианта берётся из соответствующих токенов `--content-*` (например, `--content-brand` для `default`, `--content-primary` для `secondary`, `--content-error` для `danger`).
- **FR-006**: `Link` MUST поддерживать состояния `default`, `hover`, `active`, `focus-visible`, `disabled`. Цвета и подчёркивание каждого состояния задаются токенами проекта; никаких hardcoded значений.
- **FR-007**: `Link` MUST поддерживать проп `disabled: boolean`. При `disabled=true` клик не вызывает `onClick` и не выполняет навигацию; визуально — цвет `--content-disabled`, курсор `not-allowed`, состояния `hover`/`active` не применяются. Для `<a>` компонент должен фактически блокировать навигацию (через удаление `href` или подавление события — решение в plan).
- **FR-008**: `Link` MUST поддерживать опциональные пропы `startIcon?: ReactNode` и `endIcon?: ReactNode`. Иконки наследуют цвет (`currentColor`) и масштабируются относительно текущего `font-size` (`1em` по высоте/ширине slot).
- **FR-009**: По умолчанию текст `Link` MUST быть подчёркнут (`text-decoration: underline`) с цветом подчёркивания, равным цвету текста (или использовать `text-decoration-color: currentColor`). Толщина и смещение подчёркивания — токены проекта (если отсутствуют — добавить, см. Assumptions).
- **FR-010**: `Link` MUST использовать только design tokens и шрифт `ONY One`; никаких hardcoded значений (цветов, размеров шрифта, отступов, толщины подчёркивания) в `.tsx`/`.module.css` компонента.
- **FR-011**: `Link` MUST принимать `className` и `style` и применять их к корневому элементу без переопределения внутренних токен-ориентированных стилей.
- **FR-012**: `Link` MUST пробрасывать `ref` на корневой DOM-элемент (`<a>` или `<button>`).
- **FR-013**: `Link` MUST принимать остальные нативные атрибуты соответствующего элемента (`<a>` или `<button>`) через rest props (например, `download`, `id`, `data-*`, `aria-*`).
- **FR-014**: Компонент MUST экспортироваться из `turbo-ui` (root) и подпути `turbo-ui/link` для tree-shaking; типы пропов экспортируются вместе.
- **FR-015**: В Storybook MUST появиться страница `Components → Link` со stories для всех вариантов и состояний (default, secondary, danger, hover-демо, focus-демо, disabled, в составе абзаца, в составе заголовка, с `startIcon`, с `endIcon`) и docs-страницей по шаблону проекта.
- **FR-016**: Docs-страница `Link` MUST соответствовать шаблону остальных компонентов: кнопки `Figma`/`GitHub` сверху, короткое (1–2 фразы) базовое описание главной функции, раздел `Подключение в проекте`, раздел `Пропсы`, раздел `Варианты`, раздел `Состояния` и опциональный раздел `Иконки`. Разделов «Доступность» и «Адаптивность» в этой странице НЕ создавать.
- **FR-017**: Компонент MUST размещаться в порядке Storybook рядом с другими типографическими/текстовыми компонентами (предложение по месту в навигации — в plan/реализации).

### Key Entities

- **Link**: типографическая ссылка/inline-действие. Атрибуты: `href`, `onClick`, `target`, `rel`, `variant`, `disabled`, `startIcon`, `endIcon`, плюс нативные атрибуты `<a>`/`<button>` через rest. Не имеет собственной высоты/padding — наследует typography из родителя.
- **Variant**: цветовая роль ссылки в тексте (`default`/`secondary`/`danger`); определяет набор токенов цветов для всех состояний.
- **Состояние**: `default`, `hover`, `active`, `focus-visible`, `disabled` — каждый со своим набором значений цвета/подчёркивания из токенов.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Разработчик может вставить `<Link href="...">…</Link>` в любой абзац/заголовок проекта, и ссылка визуально становится частью текста: `font-family`, `font-size`, `line-height` совпадают с родителем 1:1 (без явных переопределений в коде потребителя).
- **SC-002**: При смене токенов цвета или семейства шрифта (`tokens.json` → CSS-переменные) внешний вид `Link` обновляется без правок кода компонента.
- **SC-003**: В коде компонента (`.tsx` + `.module.css`) — 0 hardcoded цветов, размеров шрифта, толщин подчёркивания; 100% визуальных значений ссылаются на токены.
- **SC-004**: В Storybook есть страница `Components → Link` с не менее чем 8 stories (default, secondary, danger, with-startIcon, with-endIcon, in-paragraph, in-heading, disabled) и docs-страницей по шаблону проекта.
- **SC-005**: Docs-страница `Link` не содержит разделов «Доступность» и «Адаптивность»; базовое описание после кнопок `Figma`/`GitHub` — 1–2 фразы без перечисления пропов/размеров/состояний.
- **SC-006**: Компонент покрыт unit-тестами (минимум): рендер с `href`/без `href`, вызов `onClick`, поведение `disabled` (нет вызова `onClick` и нет навигации), варианты применяют соответствующие классы/токены, `ref` указывает на корневой DOM-элемент, `className` мержится без потери внутренних классов. ≥ 8 тестов.
- **SC-007**: В сборках проекта (`typecheck`, `test`, `build:lib`) после добавления компонента нет регрессий: тесты зелёные, типы валидны, билд содержит entry `turbo-ui/link`.
- **SC-008**: Компонент `Link` зарегистрирован в `storySort.order` в `.storybook/preview.js` рядом с другими текстовыми/типографическими компонентами.

## Assumptions

- **Набор вариантов** — `default` / `secondary` / `danger` — минимально достаточный по аналогии с Kontur button-link (use=default | success | danger). Финальные имена и сопоставление с токенами `--content-*` фиксируются в plan; добавление состояний `success`/`warning` — не в этой фиче.
- **Подчёркивание по умолчанию включено** для всех вариантов в состоянии покоя. Поведение подчёркивания в `hover` (исчезает / меняет цвет) — фиксируется в plan; по умолчанию — сохраняется с цветом, синхронным цвету текста.
- **Корневой элемент**: с `href` — `<a>`, без `href` — `<button type="button">`. Пропа `as`/`asChild` нет — упрощение API; при необходимости расширения — отдельной задачей.
- **Иконки**: размер `1em` (квадратные), цвет — `currentColor`. Используется существующий компонент `Icon` проекта; форма передачи (ReactNode) — как у `Button` и `IconButton`.
- **Шрифт**: компонент НЕ задаёт `font-family` явно, а наследует от родителя. Внутри `turbo-ui-scope`/`TurboUIProvider` шрифт уже `ONY One` (через токены), что выполняет требование «использовать только стили проекта и `ONY One`».
- **Доступность и адаптивность**: специальных разделов в спеке/документации нет (по требованию). Минимально необходимая семантика обеспечивается выбором корневого элемента (`<a>`/`<button>`) — это не описывается отдельным разделом, а следует из FR-002.
- **`focus-visible`-стиль** реализуется через токен фокус-кольца проекта (если такого токена нет — добавить в `tokens.json` при реализации).
- **Никаких `block`-ссылок и `button-like` стилей**: `Link` — это всегда inline-typography, без собственной высоты/padding; ссылки-кнопки (с фоном/рамкой) делаются через `Button` с подходящим `variant`, а не через `Link`.
- **Position in Storybook**: добавляется в `storySort.order` рядом с типографическими сущностями (`Foundations → Typography` или `Components → Link` после `IconButton`); финальное место — в plan.
