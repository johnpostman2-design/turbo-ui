# Data Model: TextArea

## Сущность: TextArea (компонент)

Представляет пользовательский ввод **многострочного текста** и опционально **сообщение об ошибке / подсказку** под полем.

### Поля (логическая модель / пропсы)

| Поле | Описание | Правила |
|------|----------|---------|
| `value` / `defaultValue` | Текст в поле | Controlled vs uncontrolled — как у нативного textarea |
| `placeholder` | Подсказка при пустом значении | Не смешивается с `value` как «фиктивное» значение |
| `size` | Размер визуала | `small` \| `medium` (обязательная поддержка по спеке) |
| `disabled` | Невозможность ввода | Блокирует поле; стили disabled из темы |
| `error` | Признак ошибки | Влияет на обводку и `aria-invalid` |
| `errorText` | Текст ошибки под полем | При наличии — показ под полем, `role="alert"` или связь через id; приоритет над `helperText` для отображения |
| `helperText` | Нейтральная подсказка под полем | Скрывается или не показывается при показе `errorText` (как InputField) |
| `id` | Идентификатор textarea | Для `aria-describedby` на helper/error |
| `className` | Доп. класс на корень | Не подменяет токенизированные стили обязательных состояний |

### Проброс нативных атрибутов

- Все разумные атрибуты **`HTMLTextAreaElement`** (имя, `rows`, `cols`, `readOnly`, `required`, `aria-*`, `data-*`, кроме конфликтующих с обёрткой) — через `Omit<React.TextareaHTMLAttributes<...>, ...>` с явным исключением `size` (как у Input для `size` DOM).

### Состояния отображения

| Состояние | Условие |
|-----------|---------|
| default | Не disabled, не error, не hover/focus (базовая обводка) |
| hover | `:hover` на обёртке, не disabled |
| focus | `:focus-within` на области поля |
| filled | Есть значение (не placeholder-only) — визуал текста primary |
| disabled | `disabled === true` |
| invalid | `error` или непустой `errorText` |

### Слот под полем

- Одна зона для **либо** `errorText`, **либо** `helperText`, **либо** зарезервированная пустая высота (как InputField) — чтобы layout не прыгал.

### Связи

- **Нет** обязательной связи с `label` в MVP (внешний `<label htmlFor={id}>` — ответственность потребителя).
- Текст ошибки: `id={errorId}`, `aria-describedby` на textarea при показе ошибки.
