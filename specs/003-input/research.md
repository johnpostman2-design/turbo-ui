# Research: Input (003-input)

**Purpose**: Разрешить технические решения для плана реализации Input.  
**Date**: 2026-03-14

## 1. Layout не прыгает при появлении helper/error

**Decision**: Резервировать фиксированную область под одну строку вспомогательного текста (helper или error). Контейнер под текст всегда присутствует в разметке; при отсутствии helperText и errorText рендерить прозрачный блок той же высоты (min-height) или один общий блок с условным содержимым.

**Rationale**: Спека требует «layout MUST не прыгать». Минимальное изменение — одна строка под полем с зарезервированной высотой; сложные анимации или многострочный текст выходят за scope.

**Alternatives considered**:
- Анимация высоты: добавленная сложность и риск дрожания; не требуется по спеце.
- Не резервировать место: запрещено спекой (FR-6).

---

## 2. Tab-навигация и endAdornment (IconButton)

**Decision**: IconButton в endAdornment рендерить с `tabIndex={-1}`, чтобы при Tab с input фокус уходил на следующий элемент страницы. Клик мыши по IconButton остаётся рабочим. При disabled Input — endAdornment также неактивен (передавать disabled в IconButton или оборачивать в контейнер с pointer-events).

**Rationale**: Спека явно требует «фокус по Tab с input MUST уходить на следующий элемент страницы, а не на IconButton» (FR-5, User Story 3). tabIndex=-1 исключает IconButton из порядка табуляции, но не мешает клику.

**Alternatives considered**:
- Оставлять IconButton в таб-порядке: противоречит спеце.
- Убирать endAdornment из DOM при disabled: усложнение; достаточно визуально и по взаимодействию отключить.

---

## 3. Приоритет error над helperText

**Decision**: При наличии errorText показывать только текст ошибки; helperText в этом случае не отображать. aria-describedby указывает на id элемента с errorText (или helperText, если error нет). Один контейнер под полем: либо helper, либо error.

**Rationale**: Спека (Edge cases): «приоритет у error». Один блок под полем упрощает разметку и резерв высоты.

**Alternatives considered**:
- Показывать оба: противоречит спеке.
- Два блока (helper + error): избыточно при приоритете error.

---

## 4. Токены для Input

**Decision**: Добавить в `tokens.json` секцию `input` по образцу `button` и `iconButton`: размеры (small, medium, large) с высотой, паддингами, размером шрифта; при необходимости — имена переменных для состояний (default, hover, focus, error, disabled). Конкретные значения взять из дизайна или задать согласованно с Button (например, высота поля близка к высоте Button того же size).

**Rationale**: Конституция требует Single Source of Truth; стили только из токенов. Наличие секции input в tokens.json и генерация CSS-переменных через gen-theme-css обеспечивают единообразие и переопределение темы.

**Alternatives considered**:
- Жёстко задать размеры в CSS модуле: нарушение конституции (hardcoded).
- Только существующие токены: возможно, если дизайн совпадает с button/typography; при отличии — своя секция input.
