# Data Model: ComboBox

## Сущность: Option (подсказка)

Тип одного элемента панели подсказок. Контракт совпадает с `ListboxOption`.

| Поле | Описание | Правила |
|------|----------|---------|
| `value` | Значение, попадающее в `value` поля при выборе | Уникально в пределах `options` (ответственность потребителя) |
| `label` | Отображаемый текст | Обязателен для стандартного рендера и фильтрации по умолчанию |
| `disabled` | Подсказка недоступна для выбора | Не выбирается мышью/клавиатурой; визуал — токены `disabled` |
| `icon` | Опциональная иконка слева | `ReactNode`; слот скрыт, если политика «без левых иконок» в Listbox |
| `id` | Стабильный id для `aria-activedescendant` | Генерируется при отсутствии |

### Связи

- Принадлежит упорядоченному списку `options` ComboBox.
- Не более одной выбранной по `value === option.value` (если `value` совпадает с одной из опций — она помечена как `selected`).

---

## Сущность: Query (запрос)

Текущая строка для фильтрации `options`. По умолчанию — равна текущему `value` (синхронно меняется при вводе).

| Поле | Описание | Правила |
|------|----------|---------|
| `text` | Содержимое поля | По умолчанию = `value`; trim + lowercase в `defaultFilterItem` |
| `length` | Длина | Не превышает `maxLength` (если задан) |

### Состояния

| Состояние | Условие |
|-----------|---------|
| empty | `text === ''` — показывается полный набор `options` |
| matched | `text !== ''` и есть хотя бы одна подсказка, проходящая `filterItem` |
| unmatched | `text !== ''` и `filterItem` не оставил подсказок — рендерится пустое состояние |

## Сущность: ComboBox (компонент)

Поле ввода + панель подсказок.

| Поле / проп | Описание | Правила |
|-------------|----------|---------|
| `options` | Массив `Option` | Может быть пустым; при пустом панель не открывается (см. research §2) |
| `value` / `defaultValue` | Текущее значение поля | Controlled / uncontrolled; строка |
| `onChange` | Изменение значения | `(next: string) => void`; вызывается при вводе, выборе пункта, очистке |
| `placeholder` | Плейсхолдер | Прокидывается в `Input`/`TextArea` |
| `disabled` | Блокировка | Поле не редактируется; панель не открывается; `aria-disabled` на input |
| `error` | Состояние ошибки | Визуал ошибки; `aria-invalid="true"` на input |
| `size` | Размер | `'small' \| 'medium' \| 'large'`; одинаковый для поля и панели |
| `width` / `maxWidth` | Ширина поля | Прокидываются на корень (как у TextArea/InputField) |
| `multiline` | Многострочный режим | Внутри — `TextArea`; иначе — `Input` |
| `borderless` | Поле без обводки | Через токены; состояния hover/focus/error без рамки |
| `startIcon` | Иконка слева в поле | Прокидывается в `Input.leftIcon`/`TextArea.startIcon` |
| `textAlign` | Выравнивание текста | `'start' \| 'center' \| 'end'` |
| `maxLength` | Ограничение длины | Прокидывается в input + применяется после `mask` |
| `mask` | Маска ввода | Строка по контракту research §7 (метасимволы `9`, `A`, `*`, литералы) |
| `clearable` | Показывать кнопку очистки | По умолчанию `true`; скрыта при `disabled` или пустом `value` |
| `clearIcon` | Замена стандартной иконки очистки | `ReactNode`; контейнер и обработчик — стандартные |
| `onClear` | Колбэк очистки | Вызывается перед `onChange('')` |
| `positions` | Расположение и выравнивание панели | Тот же тип, что в Select (значение или массив; fallback — первый элемент) |
| `menuOffset` | Сдвиг панели | Аналог `menuOffset` в Select |
| `menuWidth` / `menuMaxWidth` | Ширина панели | Аналогично Select |
| `menuMaxHeight` | Высота списка подсказок | Прокидывается в `Listbox.maxHeight` |
| `filterItem` | Правило фильтрации | `(query: string, option: Option) => boolean`; default — подстрока без регистра |
| `highlightMatch` | Подсветка совпадающей части | `boolean`; включает встроенную подсветку (research §6) |
| `renderOption` | Кастомный рендер подсказки | `(option, ctx) => ReactNode`; при заданном `renderOption` встроенная подсветка отключается |
| `open` / `defaultOpen` | Раскрытие панели | Controlled / uncontrolled |
| `onOpenChange` | Колбэк раскрытия | `(next: boolean) => void` |
| `name` / `form` / `required` / `autoComplete` | Нативные атрибуты | Прокидываются в `<input>`/`<textarea>` |
| `id` | Идентификатор поля | Если не задан — генерируется через `useId` |
| `className` | Класс корня | Не ломает CSS-модули |

### Связи

- ComboBox **использует** `Input` (или `TextArea` при `multiline`) для поля.
- ComboBox **использует** `Listbox` для панели; рендер — через `FloatingPortal` в `document.body`.
- ComboBox **переиспользует** механизм позиционирования из `Select` (`positions`, `menuOffset`, `flip`, `shift`, `size`, `autoUpdate`).

## Состояния отображения

### Поле (input/textarea)

| Состояние | Условие |
|-----------|---------|
| default | Не focus, не hover, не error, не disabled |
| hover | Указатель над полем |
| focus | Поле в фокусе |
| filled | `value` непустое |
| error | `error === true`; визуал ошибки + `aria-invalid` |
| disabled | `disabled === true`; визуал disabled, нет открытия |
| borderless | Модификатор без обводки; состояния выражены через фон/линию |

### Панель (listbox)

| Состояние | Условие |
|-----------|---------|
| closed | `open === false` — не рендерится |
| open / empty | `open === true`, `options` пуст или после фильтра нет совпадений |
| open / matched | `open === true`, есть подсказки |

### Иконка очистки

| Состояние | Условие |
|-----------|---------|
| visible | `clearable && value !== '' && !disabled` |
| hidden | иначе |

---

## Переходы

- `focus`/`mousedown` на поле → если `!disabled` и `options.length > 0` → `open = true`.
- Ввод символа → `value`/`query` изменяется → панель открыта; `filterItem` пересчитывается.
- Выбор подсказки (мышь / Enter с активным пунктом) → `value = option.value` (или `option.label`, см. контракт) → `onChange` → `open = false` → фокус остаётся в поле.
- Свободный ввод + `Esc`/клик-вне → `open = false`, `value` сохраняется как введено.
- Клик на иконку очистки → `onClear?.()` → `onChange('')` → `value = ''` → `open = true` (с полным набором) → фокус остаётся в поле.
- `disabled` стал `true` → `open = false`; ввод/клики игнорируются.

---

## Валидация и граничные случаи

- Две подсказки с одинаковым `value` — не валидируется библиотекой; поведение неопределённо (ответственность потребителя).
- Пустой `options` — панель не открывается; поле остаётся редактируемым; иконка очистки и подсчёт `maxLength` работают.
- `mask` несовместима с подставляемым из списка значением → `value` подсказки **проходит ту же маску перед записью**; если результат пустой/невалидный — записывается ровно как `option.value` без падения, ошибка в консоль не выводится.
- При `error === true && disabled === true` — приоритет визуала за `disabled`, `aria-invalid` остаётся `true` (для скринридера) — это совпадает с поведением Input.
- `multiline` + длинная подсказка в Listbox — подсказка остаётся **однострочной** (правила обрезки — у Listbox).
- Длинное подставляемое значение в однострочном режиме — поле прокручивается по горизонтали; `value` сохраняется полностью.
