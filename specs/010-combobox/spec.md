# Feature Specification: ComboBox

**Feature Branch**: `010-combobox`
**Created**: 2026-05-10
**Status**: Draft
**Input**: ComboBox — поле ввода с выпадающим списком подсказок. Пользователь может открыть полный список значений из справочника по клику на поле (до начала ввода) и выбрать нужное; при наборе текста список динамически фильтруется. Компонент строится поверх существующих **Input**, **Listbox** и шаблона позиционирования **Select**. Реализация — по конституции Turbo UI: только токены проекта и шрифт **ONY One**. Документация — по шаблону библиотеки в Storybook.

## Purpose

ComboBox объединяет свободный ввод и выбор из набора подсказок: пользователь либо берёт готовое значение из списка, либо печатает своё. Цель — единый предсказуемый паттерн «поле + подсказки» для форм Turbo UI, без локальных надстроек поверх Input и без дублирования API уже готовых компонентов (Listbox, Select).

## Scope

- Публичный компонент **ComboBox** с отдельным entry; внутри использует **Input** (поле), **Listbox** (выпадающая панель) и общий механизм позиционирования (как в **Select**); существующие публичные API не ломаются.
- Поле ввода с `value` / `onChange` / `placeholder` и пробросом нативных атрибутов `<input>` (форма, имя и т. п.).
- Выпадающий список подсказок:
  - открывается **по клику/фокусу** на поле — **до** начала ввода (полный набор);
  - при вводе подсказки **фильтруются динамически**;
  - выбор подсказки подставляется в поле и закрывает список.
- Конфигурация поля: размер `small` / `medium` / `large`, ширина, многострочный режим, выравнивание текста, иконка слева, поле без обводки.
- Конфигурация панели подсказок: высота списка, расположение (`positions`) и выравнивание (как у Select).
- Ввод: ограничение длины (`maxLength`), маска ввода, очистка поля кнопкой-крестиком (с возможностью заменить иконку очистки).
- Состояния: блокировка (`disabled`), ошибка (`error`).
- Подсветка совпадающей части текста в подсказках при поиске (кастомизируемая).
- **Вне scope** этой фичи: мультивыбор (теги), асинхронная загрузка подсказок с пагинацией, виртуализация очень длинных списков, валидация формата (вне маски), составное поле с лейблом/хелпером снаружи (как отдельный Field).

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Открыть полный список и выбрать значение (Priority: P1)

Пользователь кликает на пустое поле комбобокса — открывается панель со **всем** набором подсказок из справочника. Стрелками или мышью выбирает нужный пункт; значение подставляется в поле, список закрывается.

**Why this priority**: основной сценарий «справочник одной строкой ввода» — без него фича не имеет ценности.

**Independent Test**: в Storybook открыть демо ComboBox с непустым набором подсказок, кликнуть на поле — видим полный список; выбрать пункт — значение в поле, список закрыт.

**Acceptance Scenarios**:

1. **Given** ComboBox с непустым `options` и пустым `value`, **When** пользователь кликает на поле, **Then** открывается панель подсказок со всеми опциями (без фильтрации).
2. **Given** открытая панель, **When** пользователь выбирает пункт мышью или Enter с активным пунктом, **Then** `value` становится равен подписи выбранного пункта (или `value` опции, если задано), панель закрывается, фокус остаётся в поле.
3. **Given** ComboBox с заданным `value`, совпадающим с опцией, **When** пользователь открывает список, **Then** этот пункт визуально активен (выделен) в списке.

---

### User Story 2 — Динамическая фильтрация при вводе (Priority: P1)

Пользователь начинает печатать в поле — список подсказок мгновенно сужается до совпадений с введённым запросом. Выбор работает так же, как в полном списке.

**Why this priority**: без фильтрации длинные справочники нечитаемы; это вторая половина основной задачи.

**Independent Test**: ввод нескольких символов в поле сокращает видимый список до подсказок, удовлетворяющих текущему правилу фильтрации; очистка ввода возвращает полный список.

**Acceptance Scenarios**:

1. **Given** открытая панель, **When** пользователь набирает запрос, **Then** в списке остаются только подсказки, проходящие фильтр (по умолчанию — подстрока в строковом названии без учёта регистра; правило переопределяется через `filterItem`).
2. **Given** запрос не находит ни одной подсказки, **When** список открыт, **Then** панель показывает пустое состояние без ошибок и не теряет фокус ввода.
3. **Given** включена опция подсветки совпадений, **When** список отфильтрован, **Then** в каждой подсказке совпадающая часть визуально выделена (вид настраивается через слот рендера или включается флагом).

---

### User Story 3 — Свободный ввод значения вне справочника (Priority: P2)

Пользователь может ввести значение, которого нет в подсказках, и продолжить отправку формы. Поле принимает любую строку; список — только подсказка, не ограничивает ввод.

**Why this priority**: ComboBox отличается от Select именно свободным вводом; без этого он становится Select-ом с поиском.

**Independent Test**: ввести строку, отсутствующую в `options`, закрыть список (Esc/клик вне) — значение в поле сохраняется, ошибка не возникает.

**Acceptance Scenarios**:

1. **Given** список открыт и нет совпадений, **When** пользователь нажимает Esc или кликает вне поля, **Then** список закрывается, значение в поле остаётся как введено.
2. **Given** выбран пункт из списка, затем пользователь продолжает редактирование, **When** меняет текст, **Then** значение в поле обновляется, список снова фильтруется по новому запросу.

---

### User Story 4 — Конфигурация поля (Priority: P2)

Разработчик настраивает внешний вид поля под форму: размер, ширину, многострочный режим, выравнивание текста, иконку слева, вариант без обводки.

**Why this priority**: ComboBox должен встраиваться в разные сценарии форм без локальных стилей в обход компонента.

**Independent Test**: матрица состояний в Storybook покрывает все размеры, ширины, выравнивания, многострочный режим и поле без обводки.

**Acceptance Scenarios**:

1. **Given** проп `size` = `small` / `medium` / `large`, **When** компонент рендерится, **Then** высота поля и типографика соответствуют токенам соответствующего размера.
2. **Given** проп `multiline`, **When** содержимое выходит за одну строку, **Then** поле растёт по высоте по правилам **TextArea** (без потери открытия списка по клику).
3. **Given** проп `borderless`, **When** компонент рендерится, **Then** у поля нет видимой обводки, состояния hover/focus/error выражены без рамки (через токены фона/тени/линии-подчёркивания, по дизайн-системе).

---

### User Story 5 — Очистка значения (Priority: P2)

Когда в поле есть значение, справа отображается кнопка-крестик, очищающая поле одним кликом. Иконка очистки заменяема. При очистке фокус остаётся в поле.

**Why this priority**: стандартное ожидание пользователя в полях ввода и в комбобоксах справочников.

**Independent Test**: ввести значение → нажать на иконку очистки → поле пустое, фокус в поле; иконка скрыта при пустом значении и при `disabled`.

**Acceptance Scenarios**:

1. **Given** в поле непустое значение, **When** пользователь нажимает на иконку очистки, **Then** значение становится пустой строкой, вызывается `onChange('')`, фокус остаётся в поле.
2. **Given** проп `clearable={false}` или поле пустое, **When** компонент рендерится, **Then** иконка очистки не отображается.
3. **Given** проп `clearIcon` с собственной иконкой, **When** в поле есть значение, **Then** показывается переданная иконка вместо стандартной.

---

### User Story 6 — Маска ввода и ограничение длины (Priority: P3)

Поле принимает шаблон маски ввода и ограничение по длине: пользователь видит подсказку формата прямо в поле, лишние символы игнорируются.

**Why this priority**: типовая часть форм со справочниками (телефоны, индексы, артикулы); снижает количество ошибок.

**Independent Test**: при заданной маске в поле допускаются только подходящие символы; при `maxLength` ввод свыше лимита блокируется.

**Acceptance Scenarios**:

1. **Given** проп `mask`, **When** пользователь вводит символы, **Then** поле форматирует их согласно маске; невалидные символы игнорируются.
2. **Given** проп `maxLength`, **When** длина значения достигает лимита, **Then** дальнейший ввод не увеличивает значение, фокус и работа подсказок не нарушаются.

---

### User Story 7 — Позиционирование панели (Priority: P3)

Панель подсказок открывается с заданной стороны и выравнивается относительно поля по тому же контракту, что Select (`positions`).

**Why this priority**: предсказуемое поведение в модалках, рядом с краями экрана и в плотных раскладках.

**Independent Test**: сменить `positions` в Storybook на узком вьюпорте — панель открывается в указанной стороне; при невозможности — возвращается к первой позиции из массива (как в Select).

**Acceptance Scenarios**:

1. **Given** одно значение `positions`, **When** список открыт, **Then** якорение и выравнивание соответствуют этому значению из набора, общего с Select.
2. **Given** массив значений `positions` и узкий вьюпорт, **When** ни одна позиция не укладывается, **Then** используется **первая** позиция из массива.

---

### User Story 8 — Состояния блокировки и ошибки (Priority: P3)

Поле в `disabled` не открывает список и не редактируется; в `error` — визуально маркируется как невалидное и пробрасывает `aria-invalid`.

**Why this priority**: согласованность с другими полями Turbo UI (Input, Select).

**Independent Test**: переключение `disabled` и `error` в Storybook показывает соответствующий визуал; при `disabled` клик по полю не открывает список.

**Acceptance Scenarios**:

1. **Given** `disabled`, **When** пользователь кликает на поле, **Then** список не открывается, ввод не принимается; визуал — токены `disabled`.
2. **Given** `error`, **When** компонент рендерится, **Then** обводка/фон поля соответствуют ошибочному состоянию; на input установлен `aria-invalid="true"`.

---

### Edge Cases

- **Пустой `options` без ввода**: при клике на поле панель не открывается (или открывается с пустым состоянием — фиксируется в плане как один однозначный вариант), поле остаётся редактируемым.
- **Запрос не находит совпадений**: панель показывает пустое состояние без ошибок; ввод и фокус не теряются.
- **Длинная подсказка**: текст обрезается с многоточием по правилам Listbox, без переноса в строку списка (если не включён мультистрочный режим — он не относится к подсказкам, только к полю).
- **Многострочное поле + панель**: при росте высоты поля панель пересчитывает положение и не «убегает» из границ привязки.
- **Маска + автоподстановка**: подставляемое из списка значение проходит маску перед записью в `value`; если несовместимо с маской — поле сохраняет введённое пользователем значение и выводит ошибку без падения.
- **Очистка при открытом списке**: после очистки список переоткрывается с полным набором (или закрывается — фиксируется в плане однозначно), фокус остаётся в поле.
- **Ошибка + disabled**: визуал `disabled` имеет приоритет (поле недоступно, ошибочный обвод не показывается до снятия `disabled`).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-1**: Система MUST предоставлять компонент **ComboBox** — поле ввода с панелью подсказок; панель использует **Listbox**, поле — **Input** (или **TextArea** при `multiline`), позиционирование — общий механизм с **Select**.
- **FR-2**: Панель MUST открываться по клику или фокусу на поле **до** начала ввода и показывать **полный** набор подсказок, переданный в `options`.
- **FR-3**: При наборе текста система MUST динамически фильтровать видимые подсказки; правило по умолчанию — подстрока в строковой подписи без учёта регистра; MUST поддерживаться переопределение через `filterItem`.
- **FR-4**: Выбор подсказки (мышь, Enter) MUST подставлять значение в поле и закрывать панель; пользователь MUST иметь возможность ввести **произвольное** значение, отсутствующее в `options`, и сохранить его как `value`.
- **FR-5**: Компонент MUST поддерживать размеры **small / medium / large** через проп `size`; высота, отступы и типографика берутся только из токенов (`--combobox-*` / `--input-*`), без произвольных значений.
- **FR-6**: Компонент MUST поддерживать пропы ширины поля (фиксированная и максимальная) и ширины панели (фиксированная и максимальная); при отсутствии явной ширины панель MUST занимать минимум ширину поля.
- **FR-7**: Компонент MUST поддерживать **многострочный** режим поля (`multiline`); открытие/закрытие панели и фильтрация работают так же, как в однострочном.
- **FR-8**: Компонент MUST поддерживать **высоту панели подсказок** (CSS-значение); при переполнении внутри панели — вертикальная прокрутка.
- **FR-9**: Компонент MUST поддерживать **расположение** панели (`positions`: значение или массив из набора, общего с Select) и **выравнивание** относительно поля; при невозможности уложить ни одну позицию — применяется **первая** из массива.
- **FR-10**: Компонент MUST поддерживать вариант **без обводки** поля (`borderless`); состояния hover/focus/error в этом варианте передаются токенами без видимой рамки.
- **FR-11**: Компонент MUST поддерживать **иконку слева** в поле (`startIcon`); слот по размеру согласован с `size`.
- **FR-12**: Компонент MUST поддерживать **выравнивание текста** в поле (`textAlign`: `start`, `center`, `end`).
- **FR-13**: Компонент MUST поддерживать **максимальную длину значения** (`maxLength`) и **маску ввода** (`mask`); ввод сверх лимита и не подходящий маске — игнорируется без ошибок.
- **FR-14**: Компонент MUST поддерживать **очистку поля** (`clearable`, по умолчанию включена при непустом значении и отсутствии `disabled`) и **замену иконки очистки** (`clearIcon`); очистка вызывает `onChange('')` и сохраняет фокус в поле.
- **FR-15**: Компонент MUST поддерживать состояния **disabled** (поле не редактируется, панель не открывается) и **error** (визуал ошибки + `aria-invalid="true"` на input).
- **FR-16**: Компонент MUST поддерживать **подсветку совпадающей части** в текстах подсказок при наборе запроса; включение и стиль — через проп (флаг `highlightMatch` и/или слот рендера элемента подсказки).
- **FR-17**: Стили MUST задаваться **только** токенами проекта и шрифтом **ONY One**; `tokens.json` — единственный источник; никаких hardcoded значений в компоненте.
- **FR-18**: Публичный API MUST быть минимально достаточным: один способ для каждой возможности; никакого дублирования API уже существующих Listbox/Select/Input.
- **FR-19**: Компонент MUST экспортироваться отдельным entry (`turbo-ui/combobox`); документация в Storybook MUST следовать **шаблону Button** (страница Docs, якорное меню, кнопки Figma/GitHub) с секциями по каждому пропу из перечня ниже.

### Документация (обязательные секции в Docs)

Тексты в документации MUST быть короткими, без повторов, понятными и для разработчика, и для дизайнера. Каждая секция описывает поведение пропа, варианты значений и видимый результат:

1. **Размер** — `size` (`small` / `medium` / `large`).
2. **Ширина комбобокса** — фиксированная и максимальная ширина поля.
3. **Многострочное поле** — `multiline`.
4. **Высота выпадающего списка** — `menuMaxHeight` (передаётся в Listbox как `maxHeight`).
5. **Расположение выпадающего списка** — `positions` (сторона открытия).
6. **Выравнивание выпадающего списка** — `positions` (выравнивание вдоль края поля).
7. **Поле без обводки** — `borderless`.
8. **Иконка слева** — `startIcon`.
9. **Выравнивание текста в поле** — `textAlign`.
10. **Максимальная длина значения** — `maxLength`.
11. **Маска ввода** — `mask`.
12. **Очистка поля** — `clearable`.
13. **Иконка очистки поля** — `clearIcon`.
14. **Состояние блокировки** — `disabled`.
15. **Состояние ошибки** — `error`.
16. **Подсветка совпадений в подсказках** — `highlightMatch` и/или `renderOption`.

### Key Entities

- **Подсказка (option)**: отображаемая подпись, значение, доступность (`disabled`), опциональная левая иконка; контракт совпадает с `ListboxOption`.
- **Список подсказок (panel)**: упорядоченная коллекция отфильтрованных подсказок, ограничение высоты, область прокрутки; рендерится через **Listbox**.
- **Поле ввода (input)**: текущее `value`, `placeholder`, `maxLength`, `mask`, выравнивание текста, иконка слева, иконка очистки; в режиме `multiline` — на базе **TextArea**.
- **Состояние открытия (open)**: открыта ли панель; controlled (`open`/`onOpenChange`) или uncontrolled (`defaultOpen`).
- **Запрос (query)**: текущая строка ввода, по которой фильтруется панель.

### Assumptions

- Поведение «значение vs подпись» совпадает с Listbox: в `value` хранится строковая форма (значение опции при выборе из списка или произвольный текст пользователя). Если потребуется отдельное хранение объекта — обсуждается отдельной фичей.
- Контракт `positions` и правило «первая позиция при невозможности» — общий с Select; код позиционирования переиспользуется, без дублирования API.
- Подсветка совпадений в подсказках реализуется без побочного HTML в `value` (только визуально в рендере подсказки).
- Многострочный режим не меняет поведение панели: подсказки — всегда однострочный Listbox.
- Подсказки `options` поступают синхронно; асинхронная загрузка с прогрессом — отдельная фича.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-1**: Разработчик может за **менее 30 минут** подключить ComboBox в форму проекта со списком подсказок, размером, очисткой и состоянием ошибки, не изменяя код библиотеки.
- **SC-2**: На страницах Docs Storybook покрыты **все 16** свойств из перечня выше; читатель находит описание поведения и пример без обращения к исходникам.
- **SC-3**: При проверке против `tokens.json` ComboBox **не использует** hardcoded значений (цвета, отступы, типографика, радиусы); шрифт во всех состояниях — **ONY One** через токены.
