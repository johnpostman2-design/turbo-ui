# Data Model: Select и Listbox

## Сущность: Option (опция списка)

Логическая модель одного пункта; в коде — тип элемента массива или child.

| Поле | Описание | Правила |
|------|----------|---------|
| `value` | Значение для `onChange` / сравнения выбора | Уникально в пределах списка (ответственность потребителя) |
| `label` | Отображаемый текст | Обязателен для стандартного рендера |
| `disabled` | Пункт недоступен | Не выбирается по клику/клавиатуре; визуал по макету |
| `icon` | Опциональная иконка слева | `ReactNode`; слот скрыт, если политика «без левых иконок» |
| `id` | Стабильный id для `aria-activedescendant` | Генерация при отсутствии |

### Связи

- Принадлежит упорядоченному списку **options** у Select или Listbox.
- Ровно один пункт может быть **selected** в сценарии одиночного выбора.

## Сущность: Listbox (компонент)

Панель списка опций; может использоваться отдельно от Select.

| Поле / проп | Описание | Правила |
|-------------|----------|---------|
| `options` | Массив опций | Альтернатива или дополнение к `children` — в контракте |
| `value` / `defaultValue` | Выбранное значение | Controlled / uncontrolled |
| `onChange` | Смена выбора | `(value: string) => void` или с generic — в контракте |
| `selectionIndicator` | Правая отметка у выбранного | Например `check` \| `none` |
| `showItemStartIcon` | Показывать слот левой иконки | Если false — не рендерить слот даже при `icon` |
| `maxHeight` / `scrollHeight` | Высота прокручиваемой области | Токены → CSS |
| `className` | Корень панели | Не ломает BEM/модули |

### Состояния отображения (айтем)

| Состояние | Условие |
|-----------|---------|
| default | Не disabled, не под курсором (или hover по макету) |
| hover | Интерактивный пункт под указателем |
| focus / active | Клавиатура или `aria-activedescendant` |
| disabled | `option.disabled` |
| selected | `option.value === value` |

## Сущность: Select (компонент)

Триггер + портальная панель с Listbox внутри.

| Поле / проп | Описание | Правила |
|-------------|----------|---------|
| `options` | Список опций | Как у Listbox |
| `value` / `defaultValue` | Выбранное значение | Одиночный выбор |
| `onChange` | Выбор пункта | После выбора панель закрывается |
| `placeholder` | Текст при отсутствии значения | FR-8 |
| `disabled` | Блокировка | Не открывать; `aria-disabled` на триггере |
| `invalid` / `error` | Ошибка | Согласовать имя с `Input`/`Radio` (`error` boolean) — в контракте |
| `positions` | Одна или несколько позиций | Тип из 12 строк; порядок = приоритет; fallback — первый элемент |
| `menuOffset` | Горизонтальный сдвиг | 0 / положительный (влево) / отрицательный (вправо) — как в спеке |
| `menuWidth` / `menuMaxWidth` | Ширина панели | См. research §5 |
| `menuMaxHeight` | Высота области опций | FR-5 |
| `search` | Показать поле поиска | С Input из библиотеки |
| `filterItem` | Фильтрация | `(query, option) => boolean` |
| `open` / `defaultOpen` | Раскрытие | Опционально controlled для продвинутых сценариев |

### Состояния Select (триггер)

| Состояние | Условие |
|-----------|---------|
| closed / open | `aria-expanded` |
| placeholder | Нет `value` |
| disabled | `disabled` |
| invalid | `error` или `invalid` |

### Переходы

- Открытие → фокус в список или в поле поиска (уточнить в реализации по a11y).
- Выбор → `onChange` → закрытие → фокус на триггер.
- Escape → закрытие без смены значения.

---

## Валидация и граничные случаи

- Два опции с одним `value` — не валидируется библиотекой; поведение неопределённо.
- Пустой `options` — пустая панель; фокус и Escape остаются рабочими.
- `filterItem` вернул пустой список — пустое тело списка; опциональный текст «Нет результатов» — в Stories/Docs.
